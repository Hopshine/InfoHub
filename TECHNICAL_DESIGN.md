# 项目技术方案总结

## 系统概述

InfoHub是一个微信公众号信息收集与智能分析系统，专注于采集大模型相关的文章并使用Claude API进行深度内容分析。

## 技术架构

```
┌─────────────────────────────────────────────────────┐
│                   用户界面层                          │
│              (命令行交互 / 未来可扩展Web)              │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   业务逻辑层                          │
│  ┌──────────────┐  ┌──────────────┐  ┌───────────┐ │
│  │  文章采集模块  │  │  内容分析模块  │  │ 报告生成  │ │
│  │  (Collector)  │  │  (Analyzer)   │  │  (Report) │ │
│  └──────────────┘  └──────────────┘  └───────────┘ │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   数据存储层                          │
│              SQLite数据库 (articles.db)              │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                   外部服务层                          │
│  ┌──────────────┐  ┌──────────────────────────────┐ │
│  │ 搜狗微信搜索  │  │    Anthropic Claude API      │ │
│  │  (数据源)     │  │      (AI分析引擎)             │ │
│  └──────────────┘  └──────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

## 核心模块

### 1. 数据采集模块 (collector/)

**功能：**
- 通过搜狗微信搜索API搜索关键词
- 抓取文章元信息（标题、作者、发布时间等）
- 获取文章完整内容
- 去重处理

**文件：**
- `wechat_collector.py`: 主采集器
- `alternative_collectors.py`: 替代方案（第三方API、Selenium等）

**挑战与解决方案：**
- 反爬机制 → 使用代理IP池、第三方API
- 验证码 → 集成验证码识别服务
- 频率限制 → 添加随机延迟、分布式采集

### 2. 内容分析模块 (analyzer/)

**功能：**
- 使用Claude API进行文章分析
- 生成文章摘要
- 提取关键词
- 文章分类（技术解读、行业动态等）
- 深度分析（观点、创新点、应用价值）

**文件：**
- `content_analyzer.py`: AI分析引擎

**技术特点：**
- 使用Claude Sonnet 4.6模型（平衡性能和成本）
- 结构化提示词设计
- 结果解析和格式化

### 3. 数据存储模块 (storage/)

**功能：**
- SQLite数据库管理
- 文章CRUD操作
- 查询和统计

**文件：**
- `database.py`: 数据库操作封装

**数据表结构：**
```sql
articles (
    id              INTEGER PRIMARY KEY,
    title           TEXT NOT NULL,
    author          TEXT,
    account_name    TEXT,
    publish_time    TEXT,
    url             TEXT UNIQUE,
    content         TEXT,
    summary         TEXT,        -- AI生成
    keywords        TEXT,        -- AI提取
    category        TEXT,        -- AI分类
    analysis        TEXT,        -- AI深度分析
    created_at      TEXT,
    updated_at      TEXT
)
```

### 4. 配置管理 (config.py)

**功能：**
- 环境变量管理
- 配置验证
- 默认值设置

**配置项：**
- API密钥
- 搜索关键词
- 采集数量限制
- 模型选择

### 5. 工具模块 (utils/)

**功能：**
- 日志记录
- 通用工具函数

**文件：**
- `logger.py`: 日志配置

## 工作流程

### 完整流程

```
1. 用户启动程序
   ↓
2. 选择操作模式
   ├─ 采集文章
   │  ├─ 遍历关键词列表
   │  ├─ 搜索相关文章
   │  ├─ 获取文章内容
   │  └─ 保存到数据库
   │
   ├─ 分析文章
   │  ├─ 查询未分析文章
   │  ├─ 调用Claude API
   │  ├─ 解析分析结果
   │  └─ 更新数据库
   │
   ├─ 生成报告
   │  ├─ 统计分析
   │  ├─ 生成Markdown报告
   │  └─ 保存到reports目录
   │
   └─ 全流程执行
      └─ 依次执行上述三步
```

## 部署方案

### 本地部署

1. 安装Python 3.8+
2. 安装依赖：`pip install -r requirements.txt`
3. 配置环境变量：复制`.env.example`为`.env`并填写API Key
4. 运行：`python main.py`

### 定时任务

**Linux/Mac (cron):**
```bash
# 每天早上9点执行
0 9 * * * cd /path/to/infoHub && python main.py
```

**Windows (任务计划程序):**
- 创建基本任务
- 设置触发器（每天9:00）
- 操作：启动程序 `python.exe`
- 参数：`C:\path\to\infoHub\main.py`

### 服务器部署

1. 使用systemd创建服务（Linux）
2. 配置日志轮转
3. 设置监控和告警
4. 定期备份数据库

## 成本估算

### API成本（Claude）

- Sonnet 4.6: $3/MTok (输入), $15/MTok (输出)
- 单篇文章分析（约2000 tokens输入，500 tokens输出）
  - 成本：约 $0.006 + $0.0075 = $0.0135/篇
- 每天分析100篇：约 $1.35/天，$40/月

### 服务器成本

- 轻量级应用服务器：$5-10/月
- 或使用本地机器，成本为0

### 第三方API（可选）

- 天行数据：¥0.01-0.05/次
- 每天采集100篇：¥1-5/天

## 快速开始

