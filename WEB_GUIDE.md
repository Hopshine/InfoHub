## ✅ Web服务器已启动！

Web管理界面已成功启动并运行在：

- **局域网访问**: http://192.168.0.106:5000

## 🎨 界面功能

### 1. 仪表盘统计

首页顶部显示4个统计卡片：
- 📚 **文章总数**: 系统中所有文章数量
- ✅ **已分析**: 已完成AI分析的文章数
- ⏳ **待分析**: 等待分析的文章数
- 🏷️ **分类数量**: 文章分类的种类数

### 2. 操作按钮

- **🔄 刷新数据**: 重新加载统计信息和文章列表
- **🤖 批量分析**: 批量分析多篇未分析的文章（需要API Key）
- **📊 分类统计**: 查看各分类的文章数量分布图表

### 3. 文章列表

- **搜索功能**: 在搜索框输入关键词，实时过滤文章
- **分类过滤**: 下拉选择特定分类查看
- **文章卡片**: 显示标题、公众号、发布时间、分析状态
- **点击查看**: 点击任意文章卡片查看详细内容

### 4. 文章详情

点击文章后弹出详情窗口，显示：
- 完整标题和元信息
- AI生成的摘要
- 提取的关键词
- 文章分类
- 深度分析内容
- 文章正文预览
- 原文链接

如果文章未分析，可以点击"立即分析"按钮进行AI分析。

## 🚀 快速开始

### 启动服务器

```bash
python web_app.py
```

服务器启动后，在浏览器中访问：http://localhost:5000

### 停止服务器

在命令行窗口按 `Ctrl+C` 停止服务器。

## 💡 使用场景

### 场景1: 浏览已有文章

1. 打开浏览器访问 http://localhost:5000
2. 查看仪表盘统计信息
3. 浏览文章列表
4. 点击文章查看详情

### 场景2: 搜索和过滤

1. 在搜索框输入关键词（如"Claude"）
2. 或使用分类过滤器选择特定分类
3. 查看过滤后的文章列表

### 场景3: AI分析文章（需要API Key）

**单篇分析**:
1. 点击未分析的文章
2. 在详情窗口点击"立即分析"
3. 确认后等待分析完成
4. 刷新查看分析结果

**批量分析**:
1. 点击顶部"批量分析"按钮
2. 输入要分析的文章数量（建议5-10篇）
3. 确认后等待批量分析完成
4. 查看分析结果统计

### 场景4: 查看分类统计

1. 点击"分类统计"按钮
2. 查看各分类的文章数量柱状图
3. 了解内容分布情况

## 🔑 配置API Key

如果要使用AI分析功能，需要配置API Key：

1. 在项目根目录创建 `.env` 文件：
   ```bash
   copy .env.example .env
   ```

2. 编辑 `.env` 文件，填入API Key：
   ```
   ANTHROPIC_API_KEY=sk-ant-api03-你的密钥
   ```

3. 重启Web服务器：
   ```bash
   # 按 Ctrl+C 停止
   python web_app.py  # 重新启动
   ```

## 📡 API接口

Web界面使用以下API接口：

- `GET /api/stats` - 获取统计信息
- `GET /api/articles?page=1&limit=20` - 获取文章列表
- `GET /api/article/<id>` - 获取文章详情
- `POST /api/analyze/<id>` - 分析单篇文章
- `POST /api/analyze/batch` - 批量分析文章

## ⚠️ 注意事项

### 开发模式

当前运行在开发模式（Debug模式），适合：
- ✅ 本地开发和测试
- ✅ 功能演示
- ❌ 不适合生产环境

### 生产部署

如需部署到生产环境，建议：
1. 使用生产级WSGI服务器（如Gunicorn、uWSGI）
2. 配置Nginx反向代理
3. 启用HTTPS
4. 设置防火墙规则

### 安全建议

- 🔒 不要在公网暴露开发服务器
- 🔑 妥善保管API Key
- 🛡️ 生产环境添加身份验证
- 📝 定期备份数据库

## 🆘 故障排查

### 无法访问页面

1. 检查服务器是否正在运行
2. 确认端口5000未被占用
3. 尝试访问 http://127.0.0.1:5000

### 分析功能不可用

1. 检查是否配置了 ANTHROPIC_API_KEY
2. 确认API Key是否有效
3. 检查账户余额是否充足

### 数据不显示

1. 确认数据库文件存在（data/demo.db）
2. 检查浏览器控制台是否有错误
3. 尝试刷新页面
